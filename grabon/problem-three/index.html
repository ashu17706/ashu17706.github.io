I have gone through grabon.in to analyze the latency issue. My findings are as below

  • Higher no of request to the http://cdn.lmitassets.com, responsible for higher Queying time for resources (As 6 connection can be there with one domain at a time) and leading to higher total page load time. As a suggestion I think we can move these images to different cdn domains to split the load. (If image sprites techniques is not being used because of some reason).
  • Utilizing HTTPS will make Grabon more secure and future proof(moving to SPDY will be easy).
  • For better mobile experiance we can have offline support with service worker, add to home screen popup and splash screen. Which give faster User Interaction. (Not directly related to latency but faster though but better UX).
